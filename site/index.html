How it works

Six steps. No setup ceremony.

Draft or paste your script

Local editor. Autosaves to this device only.

Improve it with AI if you want

Bring your own OpenAI-compatible key. Eleven targeted rewrite actions. Nothing is sent until you click Send to AI provider.

Open Camera Lock Mode

Pick a placement preset. A small borderless window opens above all other apps.

Place it near your webcam

Drag the top strip. Reading caret sits within an inch of your camera lens.

Share only your presentation or app window

Not your full desktop. The in-app banner reminds you. Capture exclusion is an extra layer.

Present naturally

Manual scroll or Voice Follow. Practice Mode for the rehearsal pass.

Private Test Build Information

A short, honest summary of what this build is and how it handles your data. Please read this before installing.

  • This is a private beta release shared with a small group of trusted testers.
  • The application is currently unsigned. Windows SmartScreen may show a warning the first time you run it.
  • The warning exists because the application has not yet been code-signed by a commercial certificate authority. This is common for early-stage software and independent developer builds.
  • The full source code is publicly available on GitHub, and you can review it before downloading.
  • The application includes no telemetry. It does not phone home, count installs, or send usage data.
  • No account creation is required. There is no sign-up, no login, and no email collected.
  • Your scripts remain on your device in the application's local storage.
  • Your AI API key remains on your device and is sent only to the AI provider you choose, only when you explicitly trigger a rewrite action.
  • Code signing is planned if the product progresses beyond private testing. This will remove the SmartScreen warning over time.

For the complete data-handling model, including the exact storage keys used and the network calls the application makes, see SECURITY.md in the repository.

Why is the source code public?

PitchPrompter AI ships unsigned and asks for microphone access and (optionally) an AI API key. Making the source public is the simplest way to let testers confirm that the application behaves the way we describe.

  • Transparency. The exact code that builds the EXE you download is on GitHub at the tagged release.
  • Privacy verification. You can grep for fetch( and confirm there is only one outbound network path (the AI rewrite call you explicitly trigger).
  • Independent review. Security-minded testers can audit the storage, microphone, and AI code paths before they trust the build.
  • Trust building. Unsigned builds from unknown publishers are reasonable to be cautious about. Open source closes the gap until code signing is in place.

Known limitations

Calling these out up front so you know what to expect.

  • Windows only. macOS and Linux builds are not produced for v0.2.
  • Unsigned binary. Windows SmartScreen will warn on first launch. Click "More info → Run anyway".
  • Capture exclusion is experimental. Honored by Teams / Zoom / Meet / modern OBS. Not honored by legacy GDI capture or hardware capture cards. Not a security guarantee.
  • Full-desktop sharing may still reveal the teleprompter. Always prefer sharing a specific application window.
  • Web Speech API accuracy varies by accent, mic quality, and ambient noise. A local Whisper provider is on the roadmap.
  • No auto-update. We'll send you the next build directly.
